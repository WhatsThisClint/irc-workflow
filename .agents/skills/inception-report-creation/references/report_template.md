# Inception Report Output Template

This is WELL Labs' actual inception report format (from the uploaded skeleton) — use these headers exactly, in this order. This is a practical program document, not an academic paper — keep it concise per section.

1. **[Project/Theme Title]** — short title capturing the research question's theme (e.g. "Participatory Irrigation Management," "Irrigation Source Attribution via Remote Sensing").

2. **BACKGROUND** — describe the area/context and the client's (e.g. ACIWRM) intervention this study sits within. 1-2 short paragraphs.

3. **RESEARCH QUESTION** — the question as given or lightly sharpened for precision, plus the one-line conceptual model summary (IV → DV, with key mediator/moderator) from Step 2 of the workflow.

4. **PROJECT SITE** — specific site(s)/district(s); if not given, propose a plausible site consistent with WELL Labs' typical geographies (e.g. Karnataka canal command areas) and flag it as an assumption.

5. **LITERATURE REVIEWED** — the argument-mode framing from Step 3: dominant framework, stated gap, one competing account, anchor sources (flagged verified vs. illustrative per `lit_review_construction.md`).

6. **STAFF INCLUDING STUDENTS PARTICIPATING IN PROJECT** — if not specified by the user, leave as a placeholder line for them to fill in (e.g. "[PM], [Research Associate], [N research interns] — to be confirmed"), don't invent names.

7. **STUDY DESIGN** — from Step 4: sample/frame and size with reasoning, identification method, respondent list or data sources, outcome/validation assessment method, confounder handling.

8. **DETAILED WORK PLAN** — the month-by-month markdown table from Step 5.

9. **ANNEX: Survey Questionnaire / Data-Extraction Protocol** — the drafted instrument from Step 6.

Keep the whole report scannable — bullets over dense prose, and the work plan as a real table.
